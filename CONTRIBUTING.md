# 🤝 Contributing Guide


효율적인 협업을 위해 이슈(Issue) 및 PR(Pull Request) 생성 가이드를 준수해주세요.

## 📋 Issue Template
저희 프로젝트는 작업 유형에 따라 4가지 이슈 템플릿을 제공합니다.
작업을 시작하기 전에 반드시 **해당하는 이슈를 먼저 생성**해주세요.

| 템플릿 종류 | 설명 | 사용 시점 |
| :--- | :--- | :--- |
| **🚀 API Implementation** | API 개발 명세서 | 백엔드 API를 새로 개발할 때 (URI, Request/Response 정의) |
| **💡 General Feature** | 일반 기능 구현 | API 외의 작업 (DB 설계, 스케줄러, 환경 설정 등) |
| **🐛 Bug Report** | 버그 제보 | 서버 에러, 장애 발생 시 로그와 함께 제보 |
| **♻️ Refactoring & Chore** | 리팩토링 및 유지보수 | 코드 구조 개선, 라이브러리 버전 업, 문서화 작업 |

---

## 🛠 작업 흐름
1. **이슈 생성**: 위 템플릿 중 하나를 선택해 이슈를 등록합니다.
2. **브랜치 생성**: 브랜치를 만듭니다. (예: `feat/login`)
3. **개발 진행**: 커밋 메시지 컨벤션을 지켜 작업합니다.
4. **PR 생성**: 작업이 끝나면 `main` 브랜치로 PR을 보냅니다. (PR 템플릿 자동 적용됨)

---

## 💬 질문 및 토론

기능 요청이나 버그 리포트가 아닌 일반적인 질문은 [Discussions](https://github.com/Project-Guideon/guideon-backend/discussions) 탭을 이용해주세요.
